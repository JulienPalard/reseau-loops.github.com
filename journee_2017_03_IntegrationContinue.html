En complément du café LoOPS du 7 mars 2017, le réseau LoOPS a le plaisir de vous proposer toute une demi-journée sur le thème de l'intégration continue.

Cette demi-journée se tiendra le lundi 27 mars 2017 de 13h30 à 18h au Proto 204.

Nous reprendrons une session d'introduction aux tests logiciels et à l'intégration continue déjà jouée à la SED Saclay en novembre 2016, avec cependant une petite variante en vous proposant au préalable une présentation d'un retour d'expérience sur la mise en place de l'intégration continue pour les projets scikit-learn/joblib avec les plateformes Travis/CircleCI/Appveyor.
Nous poursuivrons ensuite par une mise en pratique sur un exemple simple (écrit en C++, Java ou Python au choix), à la fois localement, et sur l'infrastructure d'intégration continue de l'INRIA (ci.inria.fr).

Les programmeurs de tous horizons sont bienvenus (ingénieurs, doctorants, chercheurs...).
La présentation et le support pour les travaux pratiques sont en Anglais, mais seront exposés en Français.

Pré-requis

- la pratique d'au moins un langage de programmation (C++, Java ou Python)
- connaissance de l'outil de versionnement git
- chaque personne devra au préalable suivre cette procédure et venir avec son ordinateur avec un compilateur (C++, Java ou Python) déjà installé dessus

Inscriptions

L'inscription est gratuite mais obligatoire pour nous permettre de préparer au mieux cette demi-journée. Veillez également à obtenir un ordre de mission (éventuellement sans frais) auprès de votre unité ou de votre tutelle.

Merci de vous inscrire via ce formulaire.

Programme

  • 13h30-13h45 Accueil et présentation de la demi-journée
  • 13h45-15h00 Présentations
    • V. Rouvreau : Introduction à l'Intégration Continue : Présentation de l'infrastructure de l'Inria (30 min).
    • L. Estève : REX: Mise en place de l'IC pour les projets scikit-learn/joblib avec les plateformes Travis/CircleCI/Appveyor (30 min).
  • 15h00-18h00 Mise en pratique à partir d'un code fourni, au choix : (2h30)
      • C++ ; Java ; Python
    But:
    • Prise en main code fourni, du build local et sur la plateforme d'intégration continue
    • Implementation de tests unitaires localement et sur l'intégration continue, mise en place d'un tableau de bord de résultats
    • Mise en oeuvre de technique d'analyse statique et couverture des tests
